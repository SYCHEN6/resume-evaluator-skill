# Step 5：生成修改版完整简历

生成一份完整的、可直接使用的修改版简历，而不是差异对比。

## 必须包含的改动

- 应用报告中所有 🔴 必须修改项
- 应用所有 🟡 建议优化项（除非用户明确跳过某项）
- 未被标记修改的章节原文保留，一字不差

## 语言

保持与原简历相同的语言（中文简历 → 中文输出；英文简历 → 英文输出）。

## 输出格式

Markdown 文档，章节顺序与原简历一致。文档顶部加修改说明块：

```
<!-- 修改说明
- [修改点1一句话说明]
- [修改点2一句话说明]
- ...（每条对应报告中一个必须修改或建议优化）
-->
```

## 核心原则

- **不虚构**：只使用原简历中真实存在的经历。若改写需要数据占位，写 `[待补充]` 并注明需要什么数据，不得自行填入数字。
- **保持语气**：匹配原简历的写作风格（正式/简洁/第一人称等）。
- **输出完整**：即使只改了3条，也要输出整份简历，让用户可以直接复制。
- **标记待填项**：若某处改写需要候选人提供的具体数字或信息，用 `[待补充：XXX]` 标注。
