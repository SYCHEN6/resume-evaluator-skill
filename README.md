# Resume Evaluator Skill

一个基于 Claude Code Skill 系统的简历评估工具，通过分析职位描述（JD），对候选人简历进行多维度打分、逐条匹配分析，并给出带有原文引用和改写示例的具体修改指导。

## 功能特性

- **基本信息概览**：自动提取姓名、学历、工作年限（总年限 + 相关领域年限）、当前职位、目标岗位
- **多维度评分**：从技能匹配、经验相关性、关键词覆盖、学历背景、简历质量五个维度综合评分（满分100）
- **JD 逐条匹配分析**：对照 JD 每条要求，标注匹配度（⭐⭐⭐/⭐⭐/⭐/缺失）并直接说明差距
- **具体修改指导**：每个修改点提供 问题说明 → 原文引用 → 建议改写 的完整对照，可直接复制使用
- **ATS 关键词优化**：找出 JD 中候选人简历缺失的 ATS 高频词，给出插入位置建议
- **生成修改版简历**：用户确认后，生成完整修改版简历（Markdown 格式，含修改说明注释块，占位符标注待补充数据）
- **面试备考题目**：按技术验证、项目追问、场景设计、风险/短板、软技能五类生成定向题目；算法题可选
- **支持中英文**：中文简历、英文简历、中文 JD、英文 JD 任意组合，评估报告始终输出中文
- **支持 PDF 简历**：Claude 直接读取 PDF，无需手动复制文本

## 适用场景

- 社会招聘（社招）开发类岗位，**不适用于校招/应届生场景**
- 目标岗位：前端、后端、全栈、移动端、算法、大模型/Agent 应用等各类软件开发方向，不限编程语言和技术栈
- 企业类型：私企、国企、外企均适用，外企英文 JD 场景下可保留英文关键词供 ATS 使用

## 使用方法

### 安装

将 `SKILL.md` 及 `references/` 目录放置到 Claude Code 的 skills 目录，或通过插件系统加载。

### 触发方式

在 Claude Code 中，满足以下任意条件时会自动触发此 Skill：

- 用户提供 JD + 简历，询问匹配度或优化建议
- 用户说"帮我看看简历"、"怎么改简历"、"这个 JD 我符合吗"
- 用户上传简历文件（PDF/文本）并附上职位描述

### 使用示例

```
用户：帮我评估一下我的简历和这个JD的匹配程度，给出优化建议。

【JD内容粘贴在这里】

【简历内容粘贴在这里，或上传PDF文件】
```

也支持先提供 JD，Skill 会自动询问简历。

## 输出格式

评估报告包含以下部分，报告结尾顺序引导两个可选后续步骤：

```
# 简历评估报告

## 基本信息概览
（姓名、学历、总工作年限、相关领域年限、当前职位、目标岗位）

## 综合评分：XX / 100
（5维度评分表格）

## JD 逐条匹配分析
（每条JD要求 → 候选人情况 → ⭐ 匹配度）

## 亮点分析
（2-4条 与JD匹配的竞争优势）

## 简历修改指导
### 🔴 必须修改（影响是否过简历筛选）
### 🟡 建议优化（提升说服力）
### 🟢 关键词补充建议

## 总结与行动优先级
```

报告完成后，依次询问：
1. 是否生成**修改版完整简历**（Markdown 格式，含修改说明注释块）
2. 是否生成**面试备考题目**（5类定向题 + 可选算法题）

## 评分权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 技能匹配 | 35% | 技术栈、工具、语言与JD要求的对齐程度 |
| 经验相关性 | 30% | 岗位职责、行业、规模与JD的贴合度 |
| 关键词覆盖 | 15% | JD高频词在简历中的覆盖率（ATS核心指标） |
| 学历背景 | 10% | 学历层次和专业是否满足JD要求 |
| 简历质量 | 10% | 成就型表述、动词开头、篇幅适当 |

## 设计原则

- **不虚构经历**：修改建议只针对真实存在的经历重新表达；JD 中有但候选人未接触的技术只列为差距，不建议写入简历（哪怕是"了解"或"接触过"）
- **诚实评分**：45分就是45分，不粉饰；差距是可修复的，不是致命的
- **高分不凑问题**：综合得分 ≥ 85 时，不强行填写 🔴 修改项，避免为格式而制造问题
- **ATS 优先**：关键词匹配往往是第一道过滤，先确保过 ATS 再优化人工阅读体验
- **转型友好**：明确区分"表述不足"（可优化）和"真实能力缺口"（需补技能），帮助判断这份 JD 是否值得投

## 文件结构

```
resume-evaluator/
├── SKILL.md               # Skill 核心指令文件（含报告格式模板）
├── README.md              # 本文档
├── references/
│   ├── revised-resume.md  # Step 5：修改版简历生成规范
│   └── interview-questions.md  # Step 6：面试备考题生成规范
└── evals/
    └── evals.json         # 测试用例（4个评估场景）
```

## 测试用例

`evals/evals.json` 包含4个测试场景：

| ID | 场景 | 说明 |
|----|------|------|
| 1 | Java高级工程师 → 国有银行 | 测试技术栈缺口识别（Spring Cloud）和国企场景 |
| 2 | Python工程师转型 → Agent工程师 | 测试转型场景处理和个人项目价值评估 |
| 3 | LLM工程师 → 外企英文JD | 测试英文JD中文报告、ATS关键词保留英文 |
| 4 | Java工程师 → 国有银行（请求生成修改版简历） | 测试完整两步流程（评估 + 生成修改版） |
